# ALAX SSH Manager 技术方案

## 1. 项目定位

ALAX SSH Manager 是一个跨平台桌面端 SSH 服务器管理工具，目标是替代传统 SSH 客户端，并覆盖常用 SFTP 文件管理工作流。第一优先级是稳定、安全、可维护；功能数量放在第二优先级。

主平台优先 Windows，架构上保留 macOS 与 Linux 兼容性。界面语言先使用中文，代码变量和文件名使用英文。

## 2. 推荐技术栈

| 层级 | 技术 | 原因 |
| --- | --- | --- |
| 桌面框架 | Tauri v2 | 原生 WebView + Rust 后端，体积小、权限模型清晰、适合本地安全工具 |
| 前端 | React + TypeScript + Vite | 生态成熟，组件化强，适合复杂文件管理器和终端标签页 |
| UI | Tailwind CSS + shadcn/ui 风格组件 | 可控、统一、适合构建现代化工具型界面 |
| 状态管理 | Zustand | API 简洁，适合服务器列表、标签页、传输队列等局部全局状态 |
| 后端 | Rust | 内存安全、适合 SSH/SFTP、文件传输、异步任务和系统集成 |
| 数据库 | SQLite | 单机应用稳定可靠，便于迁移和备份 |
| 凭据 | 系统凭据存储 | 密码、私钥 passphrase 不进入 SQLite，不进入日志 |
| 终端 | xterm.js | Web 终端事实标准，支持尺寸自适应和现代主题 |
| SSH/SFTP | Rust SSH/SFTP 库封装 | 后端统一管控连接、日志、错误和敏感信息生命周期 |

## 3. 核心设计原则

1. 敏感信息只进入系统凭据存储，SQLite 只保存引用 ID。
2. SSH、SFTP、文件传输都由 Rust 后端执行，前端只负责交互和展示。
3. 大文件传输使用异步任务和传输队列，避免 UI 阻塞。
4. 所有后端错误转成结构化错误码与可读中文提示。
5. 日志默认脱敏，不记录密码、私钥、passphrase。
6. 数据表从第一阶段就按迁移思路设计，避免后续重写。

## 4. 完整功能拆分

### 服务器与分组

- 服务器 CRUD
- 分组 CRUD
- 未分组服务器
- 标签、备注、最后连接时间、连接状态
- 搜索、分组筛选、标签筛选
- 复制服务器配置
- 测试连接

### 凭据安全

- 密码凭据保存
- 私钥路径引用
- 私钥内容保存策略
- passphrase 保存
- 凭据引用 ID 与服务器记录关联
- 导入导出时默认排除敏感信息

### SSH 终端

- 多标签页
- 同一服务器多个终端
- 重连、关闭、复制、粘贴
- xterm.js 主题
- 终端尺寸同步
- 连接错误提示

### SFTP 文件管理

- 目录浏览
- 面包屑与地址栏
- 文件列表、图标、大小、时间、权限、所有者、类型
- 双击进入目录
- 新建、删除、重命名、移动、复制
- 上传、下载、拖拽上传
- 隐藏文件开关
- 文件搜索
- 右键菜单

### 传输队列

- 队列列表
- 进度、速度、状态
- 取消、重试
- 批量结果
- 错误详情

### 设置与日志

- 主题设置
- 日志目录入口
- 日志级别
- 关于页面
- 本地日志按日期滚动

## 5. 开发策略

第一阶段只完成项目基础、模块边界和 UI 骨架，不伪装已经完成真实 SSH/SFTP。第二阶段开始落地服务器/分组 CRUD 和安全凭据接口；第三阶段再接入真实 SSH 终端；第四阶段实现 SFTP 文件管理；第五阶段强化传输队列和稳定性；第六阶段做打包与视觉细节。
