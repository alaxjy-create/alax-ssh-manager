# ALAX SSH Manager 功能与安全审计

审计版本：v0.5.0  
审计日期：2026-07-10

## 结论

v0.5.0 已覆盖日常 SSH 管理和 SFTP 文件管理的核心工作流。它不追求逐项复制所有成熟商业客户端，而是把直接影响连接安全、终端可用性、文件完整性和常用运维效率的能力补齐。

## 安全与逻辑修复

| 风险 | 修复 |
| --- | --- |
| 无条件接受 SSH 主机密钥 | 改为持久指纹信任，未知密钥需确认，变化时默认阻断 |
| SQLite 外键只在初始化连接生效 | 所有数据库连接统一启用外键和 5 秒 busy timeout |
| 服务器更新可能产生跨提交不一致 | 新增、更新、删除改为事务操作，指纹清理与资料更新一起提交 |
| 新凭据保存后数据库失败会遗留凭据 | 失败时删除新引用；成功替换后清理旧引用 |
| 远程命令无退出码也被视为成功 | 只有明确收到退出码 0 才成功 |
| 命令或预览可无限占用内存 | 增加超时、16 MB 命令输出、2 MB 编辑和媒体预览上限 |
| 监控 effect 反复创建造成高 CPU | 使用稳定引用、禁止重叠请求、隐藏窗口暂停、手动启动 |
| 切换视图后终端画面变空白 | 每个会话保留独立 xterm 实例，切换时只隐藏并在恢复时强制重绘 |
| 完成的传输一直被计为活动任务 | 增加显式任务生命周期，完成时释放活动配额 |
| 运行中的传输可被重复重试 | 活动状态下拒绝重复启动 |
| 日志内容可换行注入或泄露敏感字段 | 控制字符归一化、敏感标记整条脱敏、长度上限 |
| 本地端口转发误暴露到局域网 | 后端强制仅允许回环监听，并限制并发连接与隧道数量 |
| 分组可形成循环父子关系 | 后端验证父链并拒绝自身或后代作为父分组 |
| 浏览器内容策略缺失 | 启用生产 CSP，PDF iframe 使用 sandbox |
| `russh` 可选 RSA 实现存在未修复侧信道公告 | 禁用 RSA feature；继续支持密码、Ed25519 和 ECDSA，待上游修复后恢复 |
| Linux XDG/Wayland 链路锁定高危 `quick-xml` | Linux 文件选择器切换为 GTK feature，从依赖锁文件移除受影响版本 |

## 高级能力边界

以下是成熟客户端常见但不属于核心 SSH/SFTP 可用性的高级能力，v0.5.0 未实现：

- Jump Host / ProxyJump 链路
- SSH Agent 和硬件安全密钥
- 远程端口转发与 SOCKS 动态转发
- 双向目录同步与变更监控
- 命令片段、批量执行和团队共享
- SCP、WebDAV、S3 等非 SFTP 文件协议

这些能力应在独立版本中加入，并分别设计安全模型和集成测试，避免把未经验证的网络入口塞进当前稳定版本。
